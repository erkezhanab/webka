# Материалы для пояснительной записки и защиты

## 1. Тема проекта

Разработка новостного web-портала на стеке MERN с авторизацией, разграничением ролей и управлением статьями.

## 2. Актуальность

Современные web-порталы должны обеспечивать быстрый доступ к контенту, безопасную аутентификацию пользователей, удобный интерфейс для авторов и корректную работу на разных устройствах. Новостной портал является показательной предметной областью для демонстрации полного цикла разработки web-приложения: от проектирования интерфейса до реализации REST API и взаимодействия с базой данных.

## 3. Цель проекта

Создать полнофункциональное MERN-приложение для публикации и просмотра новостных материалов с поддержкой ролей пользователей, локализации и адаптивного интерфейса.

## 4. Задачи проекта

1. Проанализировать предметную область новостных web-порталов.
2. Выполнить проектирование клиентской и серверной архитектуры.
3. Реализовать регистрацию, вход и JWT-аутентификацию.
4. Реализовать CRUD-операции для статей.
5. Реализовать поиск, фильтрацию и просмотр статей.
6. Добавить локализацию на русский и казахский языки.
7. Обеспечить адаптивный интерфейс и удобную навигацию.
8. Провести тестирование основных сценариев.

## 5. Объект и предмет разработки

- Объект: web-приложения для публикации новостей и статей.
- Предмет: разработка MERN-портала с ролью автора, администратора и читателя.

## 6. Обоснование выбора стека MERN

- `MongoDB` подходит для хранения пользователей и статей в документной модели.
- `Express` предоставляет удобный способ создания REST API.
- `React` позволяет строить динамический и компонентный интерфейс.
- `Node.js` объединяет серверную разработку на JavaScript и ускоряет реализацию полного цикла.

## 7. Архитектура проекта

### Клиентская часть

- SPA на React и Vite.
- Маршрутизация через React Router.
- HTTP-запросы через Axios.
- Хранение токена и пользовательских настроек в `localStorage`.
- Интерфейс поддерживает темы и локализацию.

### Серверная часть

- REST API на Express.
- Подключение к MongoDB через Mongoose.
- Аутентификация по JWT.
- Пароли хранятся в захешированном виде с использованием `bcryptjs`.

## 9. Основные пользовательские сценарии

### Читатель

- открывает главную страницу;
- просматривает список статей;
- фильтрует статьи;
- открывает полную статью.

### Автор

- регистрируется;
- входит в систему;
- создаёт статью;
- редактирует свою статью;
- удаляет свою статью.

### Администратор

- входит в систему;
- просматривает все статьи;
- редактирует и удаляет любые материалы.

## 10. Клиентские маршруты

- `/` — главная страница
- `/articles` — список статей
- `/article/:id` — просмотр статьи
- `/login` — вход
- `/register` — регистрация
- `/create` — создание статьи
- `/edit/:id` — редактирование статьи
- `/profile` — профиль
- `*` — страница 404

## 11. API маршруты

### Auth

- `POST /api/auth/register`
- `POST /api/auth/login`

### Articles

- `GET /api/articles`
- `GET /api/articles/:id`
- `POST /api/articles`
- `PUT /api/articles/:id`
- `DELETE /api/articles/:id`

## 13. План ручного тестирования

### Аутентификация

- регистрация валидного пользователя;
- ошибка при пустых полях;
- ошибка при некорректном email;
- ошибка при коротком пароле;
- запрет повторной регистрации на один email.

### Статьи

- создание статьи автором;
- запрет создания статьи читателем;
- редактирование своей статьи;
- запрет редактирования чужой статьи;
- удаление статьи;
- отображение статьи после создания;
- 404 при открытии удалённой статьи.

### Интерфейс

- поиск статей;
- фильтрация по категории;
- смена темы;
- смена языка;
- корректная работа в Chrome и Firefox;
- корректное отображение на мобильном устройстве.
